# Adobe I/O整合概觀

對Places REST API的每個請求都需要下列項目：

* 組織ID
* API金鑰
* 持牌代號

與Adobe I/O的整合提供這些項目，以及使用JSON網頁Token(JWT)來請求不記名Token的方式。

## 其他資訊

* 如需JWT的詳細資訊，請參 [閱JSON Web Token簡介](https://jwt.io/introduction/)。
* 若要建立Places的整合，請參 [閱建立Places整合](/help/places-rest-apis/adobe-i-o-integration/create-a-places-integration.md)。
* 若要瞭解API金鑰整合、產生JWT和公開金鑰憑證，請參閱 [Adobe I/O驗證概觀](https://www.adobe.io/apis/cloudplatform/console/authentication/gettingstarted.html)。

>[!IMPORTANT]
>
>如果您無法登入Adobe I/O主控台，或者如果「建立整合」頁面上不提供Experience Platform Location Service，請參閱「組 *織需求」*[](/help/places-rest-apis/organizational-requirements.md)。
